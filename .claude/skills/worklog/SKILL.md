# Worklog — 작업 기록 스킬

## 역할

이 스킬은 **작업 기억 관리자**다.
유저(dyl)가 Claude와 함께 noado 프로젝트를 개발하면서 나눈 대화, 내린 결정, 완료한 작업을 CLAUDE.md에 자동으로 기록한다.
기록은 다음 세션에서 맥락을 빠르게 복원하는 데 쓰인다.

---

## 실행 절차

### 1. 이번 세션 파악
현재 대화에서 아래 항목을 추출한다:
- **완료한 작업** — 실제로 코드/DB/설정이 바뀐 것
- **결정 사항** — "이렇게 하기로 했다"는 설계·방향 결정
- **유저가 언급한 중요 사항** — 유저가 강조한 조건, 주의사항, 비즈니스 룰
- **미완료·다음 할 일** — 아직 못 끝낸 것, 다음 세션에 이어야 할 것

### 2. CLAUDE.md 읽기
```
Read /Users/dyl/antigravity/noado/CLAUDE.md
```

### 3. 기록 작성 및 업데이트

CLAUDE.md 상단 "✅ 완료된 작업" 섹션에 새 항목을 추가하고,
"🚀 다음 세션 시작 순서" 섹션을 현재 상태 기준으로 갱신한다.

새 항목 형식:
```markdown
### [번호]. [작업 제목] ([날짜])
- **완료:** [완료한 것들 bullet]
- **결정:** [내린 결정들 bullet]
- **유저 언급:** [유저가 강조한 것들 bullet]
- **다음:** [이어서 해야 할 것들 bullet]
```

### 4. $ARGUMENTS 처리
유저가 인수로 메모를 전달했으면 (`$ARGUMENTS`가 비어있지 않으면),
해당 내용을 "유저 메모" 항목으로 추가한다.

### 5. 완료 후 보고
기록을 마치면 유저에게 한 줄로 보고한다:
> ✅ 작업 기록 완료 — [요약 1줄]

---

## 자동 실행 트리거 (Claude가 판단)

Claude는 아래 상황에서 **유저에게 묻지 않고 자동으로** 이 스킬을 실행한다:

| 상황 | 예시 |
|------|------|
| 주요 기능 완료 | 빌드 성공, 마이그레이션 실행, 페이지 재작성 완료 |
| 버그 수정 완료 | "이제 동작함" 확인 |
| 설계 결정 | "이 방식으로 하기로 함", 아키텍처 변경 |
| 데이터 변경 | DB 스키마 변경, 실데이터 임포트 |
| 세션 마무리 | 유저가 "끝", "수고", "나중에" 등 종료 신호 |

---

## 기록 원칙

- **사실만** 기록한다. 추측하지 않는다.
- **유저가 한 말**은 따옴표 없이 요약해서 기록한다.
- **코드 변경**은 파일명과 핵심 내용만 기록한다 (코드 전체 X).
- **과거 기록은 수정하지 않는다.** 새 항목만 추가한다.
- 날짜는 `currentDate` 컨텍스트를 우선 사용하고, 없으면 시스템 날짜를 쓴다.
