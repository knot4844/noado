# 프로젝트 개요: 노아도(Noado) - 임대 관리 SaaS

## 1. 서비스 정의 및 목표
- **서비스 도메인:** www.noado.kr
- **타깃 고객:** 50호 미만 소규모 개인 임대 사업자 (꼬마빌딩, 고시원, 오피스텔 등)
- **핵심 가치:** 엑셀과 수작업에 의존하던 임대료 수납, 독촉, 세무 일정을 100% 자동화하여 임대인의 시간을 절약하고 현금 흐름을 개선.

## 2. 기술 스택 (Tech Stack)
- **프론트엔드/백엔드:** Next.js (App Router 기반)
- **스타일링 & UI:** Tailwind CSS, shadcn/ui, Lucide React (아이콘)
- **데이터베이스 & 인증:** Supabase (PostgreSQL)
- **인프라 & 배포:** Vercel (서버리스 아키텍처)
- **외부 연동 API:** - 결제: PortOne V2 웹훅 + KG이니시스 (고정식 가상계좌)
  - 알림톡: Solapi (카카오 알림톡 자동화)

## 3. 핵심 기능 명세
- **로그인:** 카카오 / 이메일 / 휴대폰 3택 (Supabase Auth 활용)
- **대시보드:** 6대 핵심 KPI + 호실 현황 테이블 + 월별 수납 차트
- **호실/입주사 관리:** 상태별 필터링, 액션 버튼, 카드형 입주사 목록
- **수납 자동화:** 포트원 V2 웹훅을 통한 실시간 매칭 및 엑셀 업로드 자동 매칭 (신뢰도 뱃지 표시)
- **전자계약:** 계약서 미리보기(Dialog/Sheet) 및 서명 상태 추적
- **알림톡 자동화:** 청구/미납/입금완료 템플릿 미리보기 및 발송 스케줄링
- **임차인 포털:** 임차인이 스마트폰으로 접속하여 청구서 확인 및 결제 (모바일 UI 최우선)
- **세무 지원:** 부가세/종소세 신고용 매출/매입 엑셀 다운로드 기능

## 4. UI/UX 디자인 시스템 가이드라인
**A. 컬러 팔레트 (Tailwind 설정 기준)**
- **Primary (메인):** `#1d3557` (딥 네이비) - GNB, 주요 액션 버튼, 사이드바, 강조 텍스트.
- **Secondary (서브):** `#a8dadc` (라이트 블루) - 활성화 탭, 보조 버튼, 차트 포인트.
- **Semantic (상태 컬러 - 엄격하게 적용할 것):**
  - **결제/서명 완료 (Paid/Success):** 녹색 계열 (`text-emerald-700`, `bg-emerald-50`)
  - **미납/연체 (Overdue/Danger):** 붉은색 계열 (`text-rose-600`, `bg-rose-50`)
  - **청구 대기 (Ready/Warning):** 노란색/회색 계열 (`text-amber-600`, `bg-amber-50`)

**B. 레이아웃 및 반응형 규칙**
- **관리자(임대인) 뷰:** 데스크톱(PC) 최우선. 넓은 데이터 테이블과 여백을 활용.
- **임차인 포털 뷰:** 모바일(Mobile) 최우선. 터치 친화적인 큰 버튼과 세로 스크롤 위주.

**C. 컴포넌트 사용 원칙 (shadcn/ui)**
- **대시보드 KPI 카드 (총 6개):** 1) 총 임대료 예상액 2) 이번 달 수납 금액 3) 공실 현황 4) 미수금 현황 5) 계약 만료 예정 6) 세금 관련 일정
  - *배치 규칙:* 데스크톱 기준 3개씩 2줄 (`grid-cols-3`), 태블릿 `grid-cols-2`, 모바일 `grid-cols-1`.
- **데이터 표 (Table):** 금액 및 숫자 데이터는 반드시 **우측 정렬(text-right)**, 일반 텍스트는 좌측/중앙 정렬.
- **오버레이 (Dialog/Sheet):** 화면 전환을 최소화하기 위해 계약서 미리보기 등은 모달이나 사이드 시트 사용.

## 5. 외부 디자인 코드(Stitch 등) 통합 규칙
- **가장 중요한 원칙:** 사용자가 스티치(Stitch) 등에서 복사해 온 UI 코드를 제공할 경우, 절대 원본 그대로 복사/붙여넣기 하지 마세요.
- **리팩토링 지시사항:**
  1. Next.js App Router 및 shadcn/ui 컴포넌트 구조에 맞게 코드를 변환(번역)하세요.
  2. 하드코딩된 색상(Hex)이나 여백은 본 가이드라인의 컬러(Primary, Secondary, Semantic) 및 Tailwind 유틸리티 클래스로 대체하여 디자인 일관성을 맞추세요.
  3. 통짜로 된 긴 코드는 유지보수가 쉽도록 기능별로 작고 독립적인 React 컴포넌트로 분리(모듈화)하세요.
  4. 인라인 스타일(`style={{...}}`)은 제거하고 모두 Tailwind 클래스로 전환하세요.
